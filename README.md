Вариант 3: Прогноз погоды
Цель
Создать мобильное Android-приложение для отображения прогноза погоды по городам с использованием данных с сервера. Пользователь сможет просматривать прогноз на несколько дней и детальную информацию по каждому дню.
Описание приложения
Приложение позволяет пользователю выбирать город из списка и получать прогноз на первый день и на следующие дни. Прогноз отображается с температурой, погодой и иконкой. В дальнейшем приложение расширяется за счёт локального хранения, работы с сервером, архитектуры MVVM, DI, анимаций и возможности push-уведомлений.

Часть 1 – Базовый функционал
Функциональные требования
Реализовать главный экран со списком городов и прогнозом на текущий день.
Реализовать CRUD-операции:
добавление города в список
удаление города из списка
для вновь добавленного города допускается генерация “случайной” погоды
Отображение прогноза включает:
температуру;
описание погоды;
иконку погодного состояния.
Отображение списка прогнозов:
RecyclerView (XML)
или
LazyColumn (Jetpack Compose).
Для отображения элемента списка использовать:
CardView (XML)
или
Card в Jetpack Compose.
Ввод данных реализовать через диалоговое окно.


Часть 2 – Фрагменты и локальное сохранение
Функциональные требования
Разделить приложение минимум на два фрагмента:
список городов и прогнозов;
детальный прогноз выбранного дня.
Реализовать навигацию между фрагментами.
Передавать данные между фрагментами через Bundle или SafeArgs (если используется Navigation Component)
Реализовать локальное сохранение городов с погодой между перезапусками приложения в SharedPreferences.
Добавить возможность отмечать город из списка в «избранное» и показывать избранные город ДО обычных в списке.
статус “избранного” также нужно сохранять в SharedPreferences.
Использовать Snackbar для подтверждения действий:
при выборе города;
при сохранении / редактировании / удалении прогноза.



Часть 3 – Работа с сервером и асинхронность
Функциональные требования
Получать прогноз с сервера через HTTP в формате JSON.
(URL задается преподавателем, появится тут позже).
Использовать Retrofit и Gson для облегчения работы с сервером.
Отображать загруженные данные погоды в интерфейсе.
Реализовать автоматическое обновление данных одним из вариантов:
Раз в минуту через: Kotlin Coroutines + delay
Обновление по кнопке «Обновить»
Или через SwipeRefreshLayout
Обрабатывать корректные и некорректные HTTP-ответы:
при ошибке отображать Snackbar.
Каждый город и прогноз на день должен иметь уникальный id (UUID или строка).
Сетевые запросы не должны выполняться на главном потоке.
Загруженные города и прогнозы объединять с локальными, обновляя их по id.

Часть 4 – Room, MVVM и Dependency Injection
Функциональные требования
Использовать Room для кэширования прогноза погоды:
текущего дня;
исторических данных по выбранным городам и дням.
Приложение должно быть реализовано по архитектуре MVVM:
ViewModel управляет состоянием данных;
UI подписывается на изменения через LiveData или Kotlin Flow.
ViewModel не должна содержать ссылок на View или Context
Настроить Dependency Injection:
Hilt или Koin для репозиториев и базы данных.
Реализовать дополнительные функции:
фильтры по дням и городам;
сортировка по дате или температуре.

Часть 5 – Продвинутые функции
Функциональные требования
Реализовать детальный экран прогноза с расширенной информацией:
влажность;
скорость и направление ветра;
атмосферное давление.
Возможность добавления городов в “избранное” и отображение их в самом верху списка.
Реализовать Push-уведомления о изменении погоды для “избранных” городов.
Добавить базовые анимации при смене города и обновлении прогноза.
Корректная обработка разрешений для интернета и уведомлений.
